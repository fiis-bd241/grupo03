## Diccionario de Datos

### Entidad: Pedido

 Representa un pedido realizado por los usuarios de origen, generados de un reunión previa, para migrar tablas específicas de una base de datos a otra.

| Atributo | Naturaleza | Valores válidos | Definición  |
| --------- | -------------------------------------------------- | ------------------- | ------------ |
| ID_Pedido |  |  | Identificador del pedido realizado |
| Prioridad |  |  | 	 |
| Estado |  |  |  |
| Fecha Pedido |  |  | Fecha en que se realizó el pedido |
| Fecha Limite |  |  | Fecha límite para completar el pedido |

#### Tabla: Prioridad

| Código | Tipo | Detalle | 
| --- | --- | --- |
| 1 | Alta | Pedidos urgentes que requieren atención inmediata|
| 2 | Media | Pedidos importantes pero que no son urgentes |
| 3 | Baja | Pedidos que pueden ser atendidos en un plazo más amplio |


#### Tabla: Estado

| Código | Tipo |
| --- | --- |
| 1 | Pendiente |
| 2 | En proceso |
| 3 | Completado |
| 4 | Cancelado |


### Entidad: Área

 Representa las áreas funcionales de la empresa interesadas en pedir un proceso de migración de datos. 

| Atributo | Naturaleza | Valores válidos | Definición  |
| --------- | -------------------------------------------------- | ------------------- | ------------ |
| ID_Pedido |  |  | Identificador del pedido realizado |

### Entidad: Migración

Representa el proceso de trasladar campos específicos de las tablas de una base de datos a otra como parte de un pedido de migración.

| Atributo | Naturaleza | Valores válidos | Definición  |
| --------- | -------------------------------------------------- | ------------------- | ------------ |
| ID_Pedido |  |  | Identificador del pedido realizado |

#### Tabla: Tecnologia

| Código | Nombre |
| --- | --- |
| 1 | Pyspark |
| 2 | Bashsrc |
| 3 | Hsqldb |
| 4 | Kafka |
| 5 | MFRM |
| 6 | Spkscal |
